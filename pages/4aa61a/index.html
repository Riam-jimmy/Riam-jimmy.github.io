# CSRF&SSRF-审计-复现测试&同源策略

# 白盒——如何找漏洞：

成功复现->有漏洞

失败->查看代码->有过滤(能绕过)->复现成功->有漏洞

失败->查看代码->有过滤(严格过滤)->复现失败->无漏洞

# 过滤类型：

来源检测:

  1. 伪造-需要在代码数据包文件固定来源
  2. 在服务端寻找能上传文件的地方，在该地方上传数据包文件

# SSRF

如何挖洞:

  1. 特定漏洞功能，以该漏洞去代码段分析审计是否符合该漏洞
  2. 特定漏洞函数，以该函数的漏洞去测试对应功能判断是否存在相应漏洞

//CheckIsRefererValid函数如下：
    function CheckIsRefererValid()
{
    global $zbp;
    $flag = CheckCSRFTokenValid();
    if ($flag && $zbp->option['ZC_ADDITIONAL_SECURITY']) {
        $flag = CheckHTTPRefererValid();
    }

    if (!$flag) {
        $zbp->ShowError(5, __FILE__, __LINE__);
        exit;
    }
}
//checkhttp函数如下：
function CheckHTTPRefererValid()
{
    global $bloghost;
    $referer = GetVars('HTTP_REFERER', 'SERVER');
    if (trim($referer) === '') {
        return true;
    }
    $s = $bloghost;
    $s = str_replace(':80/', '/', $s);
    $s = str_replace(':443/', '/', $s);
    if (stripos($referer, $s) === false) {
        return false;
    }

    return true;
}

# SSRF

案例演示模板:yzmcms

此漏洞比较鸡肋，需要登陆进入到后台才能利用，现实中比较局限

进入管理员后台->模块管理->采集管理

构造一个网页，里面写入一个探查服务器内部文件的地址或请求一个远程网站的地址。

然后点击添加节点，项目名任意填写，网址配置填写你的攻击网页地址。

例如：x.html代码为：

<test><a href='file://d:/123.txt'></a></test>
1

此时保存后，点击采集即可获取到d盘下的123.txt的内容

yzmcms对采集网站进行测试的部分源码如下:

	public static $url;
	
	/**
	 * 获取目标网址HTML源码
	 * @param $url 目标网址url
	 * @return string
	 */
    public static function get_content($url) {
		
		self::$url = $url;
		
        $content = '';
        if (extension_loaded('curl')) {
            $ch = curl_init();
            curl_setopt($ch, CURLOPT_URL, $url);
            curl_setopt($ch, CURLOPT_RETURNTRANSFER, 1);
            curl_setopt($ch, CURLOPT_FOLLOWLOCATION, 1);
			curl_setopt($ch, CURLOPT_SSL_VERIFYPEER, FALSE); 
			curl_setopt($ch, CURLOPT_SSL_VERIFYHOST, FALSE);
            curl_setopt($ch, CURLOPT_HEADER, 0);
            $content = curl_exec($ch);
            curl_close($ch);
        } else {
            $content = @file_get_contents($url);
        }
        return trim($content);
    }


//file_get_contents()该函数是用于把文件的内容读入到一个字符串中的首选方法
//此函数的漏洞是如果$url是用户可控的，可以替换内部 IP 地址并使用您的应用程序作为代理来探测防火墙后面的网络.例如，如果他们将 $url 中的主机设置为 192.168.123.1等，这样就形成了SSRF。
